MirrorMigrationMonitor

@SPI
open class MirrorMigrationMonitor

Directs and oversees the process of MirrorMigration.

There are three responsibility of a monitor:

  1. Dictates a batch size which is a number of records processed within a single request.
  2. When the processing of a current batch is completed, a monitor is asked whether to continue or terminate.
  3. Receives notifications about the key stages in the course of the migration.

This class is open for extending. It may come in handy in the next cases:

  1. When an application environment imposes restrictions on a session duration. Which means, the migration can't be completed in a single run. For example, one can terminate the migration after the desired time since start. And then run it again to continue.
  2. When there is a need to collect data about the course of the migration. One can measure speed of the whole process or for every batch. And be aware about the number of already migrated mirrors.
