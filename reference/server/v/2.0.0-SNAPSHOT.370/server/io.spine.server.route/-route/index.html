Route

@Retention(value = RetentionPolicy.RUNTIME)
@Target(value = ElementType.METHOD)
annotation class Route

Annotates a static method of an Entity class for arranging message routing for this class of entities.

The method must accept a SignalMessage as the first parameter, and may contain corresponding MessageContext as the second parameter.

The method must return one identifier of type <I> for Unicast dispatching, and may return an Iterable<I>, if this message can be dispatched via multicast.

When used in Java, the method must be either package-private or protected for being accessible from the generated code in the same package.

The protected modifier should be used only in the very rare cases of dealing with entity class hierarchies.

When used in Kotlin, the methods must be internal, with the @Route annotation of a companion object function.

Using the annotated functions, Spine Compiler will generate classes implementing the RoutingSetup interface. Repositories will use these classes for configuring their routing schemas.
