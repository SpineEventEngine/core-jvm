ProcessManagerRepository

abstract class ProcessManagerRepository<I : Any, P : ProcessManager<I, S, *>, S : ProcessManagerState<I>> : SignalDispatchingRepository<I, P, S> , EventProducingRepository(source)

The abstract base for Process Managers repositories.

Three per-repository settings tune the optional machinery:

  • recordEventHistory — journals the events emitted by the process managers into the per-entity event journal. It is off by default: the emitted events are durable in the EventStore regardless, and the per-entity journal is a deliberate diagnostics and history index. While the journaling is off, reading the event history of a managed process manager fails fast.

  • useDoubleDispatchGuard — enables the history-backed double-dispatch guard, which rejects a signal already seen among the last eventHistoryDepth dispatches. It is off by default, and requires the event journal it scans — enable it together with recordEventHistory.

  • recordStateHistory — records the state history of the process managers on each dispatch. This is an opt-in feature shared by all entity repositories (see io.spine.server.entity.AbstractEntityRepository). It is off by default.

Type Parameters

I

The type of IDs of process managers.

P

The type of process managers.

S

The type of process manager state messages.
