HistoryStorage

abstract class HistoryStorage<I : Any, K : Any, V : Message> : MessageStorage<K, V> (source)

An abstract base for the storages of a per-entity history.

A history storage keeps items of the type V — e.g., the events emitted by an entity, or the records of its past states — appended as the entity handles its signals. Each stored item exposes the three columns, allowing to manage the history and query it efficiently: the packed identifier of the entity, the time the item was created, and the number of the entity version the item belongs to.

On top of them, the storage provides the window reads ordered from newer to older, and the time-based truncate maintenance operation.

While the reads and the maintenance operations are open to applications, new kinds of histories cannot be defined outside the framework: the constructor is internal. Storage vendors customize the persistence via the RecordStorage delegate their StorageFactory creates in createRecordStorage.

Parameters

context

Specification of the Bounded Context in the scope of which the storage is used.

spec

The specification of the history storage: the record spec persisting the items (which must list the history columns), the columns to query the history by, and the storage group telling this history apart from the latest-state storage of the same entity.

factory

The storage factory to use when creating a record storage delegate.

Type Parameters

I

The type of the entity identifiers.

K

The type of the history item identifiers.

V

The type of the stored history items.
