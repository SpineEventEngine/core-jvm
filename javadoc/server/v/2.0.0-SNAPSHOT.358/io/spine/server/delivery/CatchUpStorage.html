    • Constructor Detail

      • CatchUpStorage

        CatchUpStorage(StorageFactory factory, boolean multitenant)

        Creates a new instance of this storage.

        It is recommended to have CatchUpStorage instances single-tenant only. It is so, because no distinction should be made for processing of InboxMessages sent during the catch-up process, since it is batch-based anyway, and splitting batches even more (across tenants) reduces the performance.

        Parameters:
        factory - storage factory to create an underlying record storage
        multitenant - whether CatchUpStorage should be multi-tenant
