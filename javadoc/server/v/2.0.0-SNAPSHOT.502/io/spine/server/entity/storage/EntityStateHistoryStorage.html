Package io.spine.server.entity.storage

Class EntityStateHistoryStorage

  • All Implemented Interfaces:
    io.spine.server.Closeable , io.spine.server.storage.Storage , java.io.Closeable , java.lang.AutoCloseable 
    
public final class EntityStateHistoryStorage<I extends Object>
extends HistoryStorage<I, EntityStateKey, EntityRecord>

    The history of recent states of entities, retained until the application trims it.

    This HistoryStorage stores the EntityRecord an entity had after each dispatch, keyed by the entity identifier and the version number. Entities read their recorded states through a StateHistoryLoader installed by the repository — e.g., AbstractEntity.stateAt(Timestamp) — and the storage also serves debugging and analysis, answering the "state at a time" query. The history is never used for loading: an entity loads from its latest record in the corresponding EntityRecordStorage.

    Recording is opt-in per repository and is off by default; see recordStateHistory in AggregateRepository. The history is not trimmed automatically: retention is the duty of the application, served by the trim and truncate maintenance operations.

    The records are stored as-is; the entity, the time the state became current, and the version number are exposed for querying as the columns derived from the record.

    Nothing in this storage is specific to a kind of entity; currently, Aggregates are the only kind recording their state history.

    The storage is identified by the served entity class paired with the type of the stored items, EntityRecord: vendors allocate the physical storage by this pair (see createRecordStorage), so a state history stays apart from the histories of other entity classes — even when their identifier values coincide — and from the latest-state records of its own entity class, which never arrive at the history seam.

    The class is deliberately final: storage vendors customize the persistence via the RecordStorage delegate created by their StorageFactory.

      • stateAt

         final EntityRecord stateAt(I entityId, Timestamp at)

        Returns the state record the entity had at the given time if the history retains it.

        The result is the retained record with the most recent timestamp not later than at. When several retained records share that instant, the one with the highest version wins.

        The answer is honest about retention: null means the question cannot be answered from the retained window — the time either precedes the oldest retained record, or predates the entity itself.

        The record is selected by a single query on the created column — the newest record of the entity at or before at — so the backend returns the answer directly, without reading the history into memory.

        The records are ordered by created before version: a backend such as Datastore requires the property under an inequality filter to lead the sort. So the answer is the newest record by creation time — the highest version as long as an entity records its states in the version order, the case for the live per-entity dispatch feeding this history. Should a higher version ever be recorded with an earlier timestamp, the result follows the timestamp rather than the version.

        Parameters:
        entityId - The identifier of the entity.
        at - The point in time to look at.
        Returns:

        The record effective at the given time, or null if the history does not retain it.

      • trim

         final Unit trim(I entityId, Integer keepMostRecent)

        Trims the history of the entity with the given identifier, keeping up to keepMostRecent most recent records. Passing zero purges the history of the entity.

        Ranks the records by an identifier-only read: the record keys of this storage carry the version, so ranking needs no record payloads — trimming reads small identifiers instead of full records with packed states.

        Parameters:
        entityId - The identifier of the entity.
        keepMostRecent - The number of the most recent records to keep.
