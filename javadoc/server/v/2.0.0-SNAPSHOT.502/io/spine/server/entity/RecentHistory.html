Package io.spine.server.entity

Class RecentHistory

  • All Implemented Interfaces:
    
public abstract class RecentHistory<R extends Object, T extends Object, L extends HistoryLoader<R>>

    A view on the recent history of an AbstractEntity, read lazily from a durable storage and cached for the lifetime of the entity instance.

    The history serves its reads through a loader, installed by the repository managing the entity. The items obtained from the loader — and the items appended by the framework as the entity produces them — are kept in an instance-scoped cache, so that an entity handling several signals in a batch does not re-read the same items from the storage on every dispatch. The cache lives and dies with the entity instance; a repeated read hits the storage only below the items already cached.

    An entity created outside a repository has no loader installed, so its reads serve only the appended items, if any.

    The instance is confined to the thread dispatching the signals of the entity, as the entity itself is; the cache is not synchronized.

    The kinds of recent histories are fixed by the framework: the constructor is internal.

      • read

         final Iterator<T> read(Integer depth)

        Reads up to depth most recent items of the history, newest first.

        The items already known to this instance — cached by the previous reads or appended — are served from memory. If more items are requested than cached, the read continues below the cached run through the installed loader, caching the traversed items on its way. If no loader is installed — the entity was created outside a repository — only the cached items are served.

        Fewer items are returned if the history retains fewer.

        Parameters:
        depth - The maximum number of the most recent items to read.
        Returns:

        An iterator over the items, newest first.
