Package io.spine.server.aggregate

Class AggregateStorage

  • All Implemented Interfaces:
    io.spine.server.Closeable , io.spine.server.storage.Storage , java.io.Closeable , java.lang.AutoCloseable 
    @SPI() 
public class AggregateStorage<I, S extends AggregateState<I>>
extends AbstractStorage<I, EntityEventHistory>

    A storage of the latest Aggregate states and the journal of the events emitted by Aggregates.

    The instances of this type solve two problems:

    • storing the latest state of each Aggregate along with its lifecycle flags and version — the source of truth for loading an Aggregate by its AggregateRepository;
    • journaling the events emitted by the Aggregates — for traceability and for the recent-history lookups such as historyBackward and the opt-in IdempotencyGuard.

    The state of an Aggregate is persisted on each update, regardless of the visibility of the Aggregate state type. Similar to storages of other Entity types, the states are persisted as EntityRecords in an intermediate EntityRecordStorage, to which all state-related operations are delegated.

    End-users of the framework are able to set the visibility level for each Aggregate state by using an (entity).visibility option in the Protobuf message corresponding to the Aggregate state. For those Aggregates that are visible, the framework routines enable this storage to expose the latest states of Aggregates for further querying. To some extent, it makes this storage a part of an application's read-side. Visibility gates only this query exposure — never the state writes or the load-path reads.

    AggregateStorage supports querying the Aggregate states by the values of their declared entity columns. See io.spine.query package docs for more details on the query language.

    The events emitted by the Aggregates are appended to an intermediate EntityEventStorage. The journal is not used for restoring the Aggregate states. It narrows down the number of records to traverse when the recent history of an Aggregate is read, compared to searching the Event Store of the whole Bounded Context.

    The journal grows as the Aggregates emit events. To bound the growth, production systems run the count- or date-based truncation as a periodic maintenance operation.

    Before the event-sourcing cutover, the journal was persisted as AggregateEventRecords — a record kind that could also hold Snapshots. The current runtime neither writes nor reads records of that kind. They remain in the underlying storage as inert data, and their proto definitions are retained so that the persisted records stay parseable.

      • index

         Iterator<I> index()

        Returns an iterator over identifiers of Aggregates served by this storage.

        The results include IDs corresponding only to those Aggregate instances whose state was written to this storage. Starting with Spine 2.0, the framework writes the essential bits of every Aggregate — its identifier, lifecycle flags, and version — to this storage on each update, regardless of the visibility of the Aggregate state. Therefore, for the data produced by the current framework version, this index returns all the Aggregates of the served type.

        The identifiers found only in the persisted events are not taken into account, as obtaining them would require scanning and de-duplicating too many event records. As a consequence, an Aggregate persisted by an older framework version is not returned until its state is written to this storage. For an Aggregate that still has a record in the now-deprecated Mirror projection, run the Mirror migration, which writes the migrated states into this storage. An Aggregate that exists only as events — with neither a state record nor a Mirror record — is picked up once its state is next written, for example, on its next update.

         void write(I id, EntityEventHistory events)

        Writes events into the storage.

        NOTE: This method does not rewrite any events, just appends them. Many events can be associated with a single aggregate ID.

        Each event is journaled under its producer — the aggregate which emitted it. For the events emitted by the aggregate with the passed identifier, the producer and the identifier match.

      • readStates

         Iterator<EntityRecord> readStates(TargetFilters filters, ResponseFormat format)

        Queries the storage for the Aggregate states according to the passed filters and returns the results in the specified response format.

        The results of this call are eventually consistent with the latest states of aggregates, as instances are not restored from their events for querying. Instead, this method works on top of the storage of the latest known aggregate states, for better performance. In a distributed environment, the records in this storage may be outdated, as new events may have been emitted by an aggregate instance on other server nodes.

        Parameters:
        filters - the filters to use when querying the aggregate states
        format - the format of the response
        Returns:

        an iterator over the matching EntityRecords

      • truncate

         void truncate(int keepMostRecent)

        Truncates the journal, keeping up to keepMostRecent most recent events for each Aggregate instance.

        Passing zero purges the whole journal of the served Aggregates.

        Truncation bounds the recent history available to the business logic and to the opt-in IdempotencyGuard. When the guard is enabled, keep at least the history depth of the repository, so that the deduplication window stays intact.

        The operation reads the whole journal, so it is intended for periodic maintenance rather than for per-dispatch use.

        Parameters:
        keepMostRecent - the number of the most recent events to keep for each Aggregate

      • truncate

         void truncate(int keepMostRecent, Timestamp olderThan)

        Truncates the journal, deleting the events older than olderThan, but keeping at least keepMostRecent most recent events for each Aggregate instance.

        An event record is deleted only if its event was emitted before the given time and it is not among the keepMostRecent most recent events of its Aggregate. To purge everything older than the given time, pass zero as keepMostRecent.

        See truncate for the notes on the IdempotencyGuard window and the performance expectations.

        Parameters:
        keepMostRecent - the number of the most recent events to keep for each Aggregate
        olderThan - only the events emitted strictly before this time are deleted
